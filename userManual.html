Installation Guideline


The program is built to be used on the http://consort.cs.ualberta.ca webserver. In order to install, the following steps should be followed:

1. Unzip the system file.

2. Copy the contents of the "code" directory to /compsci/webdocs/<yourccid>/web_docs/OceanObservationSystem/

3. Set the permissions for all of the files to 644 (read only for others) except for scalarid.txt, download.php, downloadscalars.php, uploadaudio.php, uploadimage.php, uploadscalars.php which needs to be set to 666 (read and write for everyone).

4. Modify line 3 of PHPconnectionDB.php to be:

$conn = oci_connect('SQLPLUS USERNAME', 'SQLPLUS PASSWORD')

Where SQLPLUS USERNAME is replaced with your sqlplus username and SQLPLUS PASSWORD is replaced with your sqlplus password.

5. Run setup.sql and then timetable.sql into sqlplus using the same account that was put into the PHPconnectionDB.php.


The system should now be functional. In order to access it open up a browser (firefox is recommended) on any university of alberta computer sciences lab computer and go to http://consort.cs.ualberta.ca/~<your-ccid>/OceanObserverSystem/index.php and log in as default administrator (u: admin, p: admin).


Login Module


The login module will always appear as the only webpage available as long as the websession is in a logged out state (no cookies are present, and all other pages redirect to the log in). Enter your username and password and select the "Log In" button to log in. The page you are redirected to is based on your user's role (scientist, data curator or administrator).






Sensor Management


The "Sensor Management" tab allows an administrator to add or remove sensors from the observation system:

  • Add Sensors:

    • 4 fields must be filled to successfully add a sensor to the database:

      • "Enter Sensor ID:" requires a number that has yet been used to signify the sensor's identification number.

      • "Enter Location:" requires the location the sensor has been placed

      • "Enter Description:" requires a brief description of what the sensor is required to perform.

      • "Type:" is a radio button selection to provide one of the three forms of sensor operation: Audio, Image or Scalar data.

    • Selecting the "Add Sensor" button will insert the new sensor into the observation system's database.

  • Remove Sensors:

    • 1 field must be filled to successfully remove a sensor to the database:

      • "Enter Sensor Id:" requires the ID number of an existing sensor.

    • Selecting the "Remove Sensor" button will delete the sensor from the database.


User Management


The "User Management" tab allows an administrator to add, delete and update users from the database:

  • Add Users:

    • Add New User:

      • 9 fields must be filled to successfully add a new user to the database:

        • "Enter new username:" requires a new username

        • "Enter new users password:" requires the new users default password.

        • "Enter new personID:" requires a unique, new person ID number

        • "Enter first name:" requires the new user's first name

        • "Enter Last Name:" requires the new user's last name

        • "Enter Address:" requires the new user's address

        • "Enter email" requires the new user's email address (NOTE: Must be a unique email address from all other persons!)

        • "Enter phone number" requires the new user's 10-digit phone number

        • "Select role:" is a radio button selection to provide one of the three forms of user: Administrator, Scientist, or Data Curator.

    • Add Existing User:

      • 4 fields must be filled to successfully add a user for an existing person into the database:

        • "Enter email" requires the existing person's email address

        • "Enter new username:" requires a new username

        • "Enter new users password:" requires the new users default password.

        • "Select role:" is a radio button selection to provide one of the three forms of user: Administrator, Scientist, or Data Curator.

  • Delete Users:

    • Delete a User:

      • 1 field required to delete a user:

        • "Enter Username you wish to delete:" requires the username of the user you would like to remove

    • Delete a person:

      • 1 field required to delete a person:

        • "Enter the email of person you wish to delete:" requires the email of the person you would like to remove. This will remove both the person and users of person from the database.

  • Update User Information:

    • Change User Password:

      • 3 fields required to change a user's password. If the new passwords match in both fields, the password will change for the user:

        • "Enter Username:" requires the username of the user that would wish to have their password changed.

        • "Enter New Password:" requires the new password

        • "Re-enter New Password:" requires the same password as the previous field.

    • Change User's Role:

      • 2 fields required to change a user's role:

        • "Enter Username:" requires the username of the user that would wish to have their role changed.

        • "Enter New Role:" is a dropdown list of roles the user can be changed to.

    • Change Person's Information:

      • 1 field required:

        • "Enter Email of Person:" requires the email of the person you would like to change their personal information. This will direct to another page to enter person's changed information.


Account Settings


The "Account Settings" tab allows all users to change their respective passwords and personal information:

  • Change Password:

    • 2 fields required:

      • "Enter New Password:" requires a new password

      • "Re-enter New Password:" requires the same password as the previous field. The password will only change if both fields have matching passwords.

  • Modify Personal Information

    • Any field with current content will display information. To modify, 5 fields are required:

      • "First Name:" requires first name

      • "Last Name:" requires last name

      • "Address:" requires address

      • "Email:" requires email address (NOTE: Must be a unique email address from all other persons!)

      • "Phone:" requires a 10-digit phone number


Subscribe

The "Subscribe" tab allows scientists to subscribe and unsubscribe from available sensors within the ocean observation system:

  • Subscriptions: A list of sensors you are currently subscribed to receive data from. Sensors will display their ID, location, sensor type, and description. Each sensor will also contain an "Unsubscribe" button. Selecting this button will unsubscribe you from the sensor, and it will appear under "Other Sensors".

  • Other Sensors: A list of sensors you are currently not subscribe to receive data from. Sensors will display their ID, location, sensor type, and description. Each sensor will also contain a "Subscribe" button. Selecting this button will Subscribe you to the sensor, and it will appear under "Subscriptions".



Search

The "Search" tab allows scientists to search the system for records provided by their subscribed sensors between time periods. The search will also allow a scientist to preview and download data from the sensors:

  • 5 fields are allowed to be inputted to specify searches:

    • "Keywords:" Any specific string of text that can appear in either the sensor's general description, or the audio or image description.

    • "Sensor types:" Selection of sensors (Audio, Image, Scalar Data, or All(.

    • "Location:" Location of sensor

    • "Start Date of Records:" required start date of record search.

    • "End Date of Records:" required end date of record search.

  • Pressing "Search Records" will send you to results page, detailing all records matching search requirements. All records will display the sensor ID, location, sensor description, specific record id, and the date created

    • Images will display a thumbnail of image records. Selecting "download" will download the image as a .jpg file.

    • Scalar Data will display the value of the data. Selecting "Downaload" will download the scalar data as a .csv file. Selecting "Downaload All Scalars" at the bottom of the table will download all combined sensor records as a single .csv file.

    • Audio will display the audio length (in seconds). Selecting "Download" will download the audio as a .wav file.


Data Analysis Report

The "Data Analysis Report" tab allows scientists to select and view scalar data's average, minimum, and maximum values within a generalized date:

  • The drop down menu displays all scalar sensors you are subscribed to. Select the sensor you would like to analyze, and select "Get Sensors Reports"

  • The next page that appears will display the sensors id, location, and separated average, minimum, and maximum values by year.

    • Selecting the Year button will drill down the scalar sensors data by each quarter of the year.

    • You can continue to drill down from year, to quarter of year, to month of quarter, to week of month, to day of week.

    • At any time you may select other buttons to re-generalized to that specific time period

    • At any time you may select the "Return to Userpage" to return to the Scientist Dashboard.

Upload Module

The "Upload" tab allows data curators to upload image files, audio files, and batch scalar csv files to the database. To upload a certain type of content click the corresponding button and fill out the required fields.

  • Upload Image

    • There are four required fields and one optional field needed to add a image

    • "Image ID:" requires a unique image Id number

    • "Sensor ID:" requires the id for an existing image sensor

    • "Date Created:" requires a date for when the image was created in the form of dd/mm/yyyy hours(24):minutes:seconds. This will be autofilled with the current time (the time the current page was loaded)

    • "Description:" is an optional field that allows you to put in a string describing the image

    • "File:" is the image file you wish to load to the database. It must be in the form of a png. If browse is clicked the user will be prompted to select a file from their file system

  • Upload Audio

    • There are four required fields and two optional fields needed to add an audio file

    • "Recording ID:" requires a unique recording Id number

    • "Sensor ID:" requires the id for an existing audio sensor

    • "Date Created:" requires a date for when the audio was created in the form of dd/mm/yyyy hours(24):minutes:seconds. This will be autofilled with the current time (the time the current page was loaded)

    • "Description:" is an optional field that allows you to put in a string describing the image

    • "Length:" is an optional field that allows you to put a numeric value that represents the length of the audio file

    • "File:" is the audio file you wish to load to the database. It must be in the form of a wav. If browse is clicked the user will be prompted to select a file from their file system

  • Upload Scalars

    • The scalar upload allows you to upload a scalar data in the form of a csv file. Each line of the given csv file will represent one scalar data recording so multiple scalar information can be uploaded at once.

    • "File:" is the csv file you wish to load to the database. If browse is clicked the user will be prompted to select a file from their file system

    • Each line in the csv file must have the form "sensor_id, date created, value" e.g For scalar data from sensor 3,. created on 01/11/2015 at 22:34:21 with a value of 4 the csv line would be "3, 01/11/2015 22:34:21,4".

    • Note the sensor Id must be for a scalar sensor


Once the submit button is clicked for any data type the user will sent to a page where the status of the upload is shown and can return to the data curator page by clicking the "return to user page" button.
